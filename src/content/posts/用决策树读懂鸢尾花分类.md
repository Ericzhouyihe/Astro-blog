# 用决策树读懂鸢尾花分类：从基尼系数到模型评估

## 一、要解决什么问题

决策树是一种非常直观的机器学习算法。它的思路很像我们日常做判断：

> 如果花瓣长度很短，那么它大概率是山鸢尾；否则继续看花瓣宽度，再继续细分。

也就是说，决策树会把一个复杂问题拆成一连串“是 / 否”判断，最后走到某个叶子节点，得到分类结果。

本文基于鸢尾花数据集，完整梳理一个使用 `sklearn.tree.DecisionTreeClassifier` 完成分类预测的流程，包括：

1. 数据加载与特征、标签拆分；
2. 标签编码；
3. 训练集和测试集划分；
4. 决策树模型训练；
5. 准确率、精确率、召回率、F1 值评估；
6. ROC 和 AUC 理解；
7. 决策树可视化结果解读。

------

## 二、鸢尾花数据集是什么

鸢尾花数据集是机器学习入门中非常经典的数据集。它的任务是：根据一朵花的 4 个测量值，预测它属于哪一种鸢尾花。

数据中的 4 个特征通常表示为：

| 特征              | 含义     |
| ----------------- | -------- |
| A 或 sepal length | 花萼长度 |
| B 或 sepal width  | 花萼宽度 |
| C 或 petal length | 花瓣长度 |
| D 或 petal width  | 花瓣宽度 |

目标类别有 3 种：

| 原始类别        | 编码后 |
| --------------- | ------ |
| Iris-setosa     | 0      |
| Iris-versicolor | 1      |
| Iris-virginica  | 2      |

可以把这个问题想象成一个植物学小助手：你告诉它花萼和花瓣的尺寸，它根据历史样本总结出来的规则，判断这朵花最像哪一种。

标准化的常见公式是：

$z = \frac{x - \mu}{\sigma}$

其中：

- $x$ 是原始值；
- $\mu$ 是均值；
- $\sigma$ 是标准差；
- $z$ 是标准化后的值。

不过，对于决策树来说，通常不强制要求标准化。
因为决策树关心的是“某个特征是否小于等于某个阈值”，例如：

$x_3 \leq 2.35$

它并不像 KNN、SVM、逻辑回归那样非常依赖特征尺度。

所以这里不做标准化也是合理的。

## 九、决策树的核心思想

决策树每次都会选择一个特征和一个阈值，把数据分成两部分，让分裂后的数据尽量“纯”。

所谓“纯”，可以理解为：

> 一个节点里最好都是同一种类别。

比如一个盒子里全是苹果，那么它很纯；
如果里面既有苹果、梨、香蕉，就不纯。

决策树常用的不纯度指标有基尼系数和信息熵。

------

## 十、基尼系数原理

代码中使用的模型默认评价指标是 `gini`，也就是基尼系数。

基尼系数公式为：

$Gini(D) = 1 - \sum_{k=1}^{K} p_k^2$

其中：

- $D$ 表示当前数据集；
- $K$ 表示类别数量；
- $p_k$ 表示第 $k$ 类样本在当前节点中的占比。

如果一个节点里所有样本都属于同一类，例如类别分布是：

$1, 0, 0$

则：

$Gini(D) = 1 - (1^2 + 0^2 + 0^2) = 0$

基尼系数为 $0$，表示节点非常纯。

如果一个节点里三类样本比较混杂，比如：

$\frac{1}{3}, \frac{1}{3}, \frac{1}{3}$

则：

$Gini(D) = 1 - ((\frac{1}{3})^2 + (\frac{1}{3})^2 + (\frac{1}{3})^2) = \frac{2}{3}$

基尼系数越大，说明越混乱；基尼系数越小，说明越纯。

------

## 十一、信息熵补充理解

虽然本模型使用的是基尼系数，但决策树中也经常使用信息熵。

信息熵公式为：

$H(D) = - \sum_{k=1}^{K} p_k \log_2 p_k$

信息熵也是衡量混乱程度的指标：

- 熵越大，数据越混乱；
- 熵越小，数据越确定；
- 如果所有样本都属于同一类，熵为 $0$。

生活化理解：
如果你闭眼从袋子里摸糖，袋子里全是苹果味糖果，你不用猜也知道摸到什么，信息熵低。
如果袋子里有很多种口味，而且比例接近，你就很难猜，信息熵高。

------

## 十二、构建决策树模型

代码中创建模型如下：

```python
algo = DecisionTreeClassifier(max_depth=3)
```

这里只设置了一个关键参数：

| 参数          | 含义                 |
| ------------- | -------------------- |
| `max_depth=3` | 限制树的最大深度为 3 |

为什么要限制深度？

如果不限制，决策树可能会把训练数据分得特别细，甚至记住每一个训练样本。这样训练集表现很好，但测试集可能表现变差，这种现象叫过拟合。

可以把它理解成：
一个学生把练习册答案全背下来了，练习题全对，但换一套新题就不会了。

限制树深度，就是让模型不要学得太死。

常见参数还有：

| 参数                | 作用                                  |
| ------------------- | ------------------------------------- |
| `criterion`         | 不纯度指标，可选 `gini` 或 `entropy`  |
| `splitter`          | 特征划分方式，可选 `best` 或 `random` |
| `max_depth`         | 最大树深度                            |
| `min_samples_split` | 节点至少有多少样本才允许继续分裂      |
| `min_samples_leaf`  | 叶子节点至少保留多少样本              |
| `class_weight`      | 类别权重，适合类别不平衡问题          |

## 十五、分类评估报告

代码分别输出了训练集和测试集的分类报告：

```python
print(classification_report(y_train, algo.predict(x_train)))
print(classification_report(y_test, algo.predict(x_test)))
```

训练集结果：

```
              precision    recall  f1-score   support

           0       1.00      1.00      1.00        36
           1       1.00      0.95      0.98        42
           2       0.95      1.00      0.98        42

    accuracy                           0.98       120
```

测试集结果：

```
              precision    recall  f1-score   support

           0       1.00      1.00      1.00        14
           1       1.00      0.88      0.93         8
           2       0.89      1.00      0.94         8

    accuracy                           0.97        30
```

几个指标的含义如下。

### 1. 准确率 Accuracy

准确率表示所有预测中，有多少预测对了：

$Accuracy = \frac{预测正确的样本数}{总样本数}$

本例中：

- 训练集准确率为 $0.9833$；
- 测试集准确率为 $0.9667$。

说明模型整体表现很好。

### 2. 精确率 Precision

精确率表示：模型预测为某一类的样本中，有多少是真的。

$Precision = \frac{TP}{TP + FP}$

其中：

- $TP$ 表示真正例；
- $FP$ 表示假正例。

生活化理解：
如果模型说“这些都是 Iris-versicolor”，精确率关心的是：这些被模型点名的样本里，有多少真的就是 Iris-versicolor。

### 3. 召回率 Recall

召回率表示：真实属于某一类的样本中，有多少被模型找出来了。

$Recall = \frac{TP}{TP + FN}$

其中：

- $FN$ 表示假负例。

生活化理解：
如果真实有 100 个目标客户，模型找出了 90 个，那么召回率就是 $90%$。

### 4. F1-score

F1 值是精确率和召回率的综合指标：

$F1 = \frac{2 \times Precision \times Recall}{Precision + Recall}$

当我们既关心“找得准”，又关心“找得全”时，F1 值很有用。

$P(Iris-setosa) = 1.0$

$P(Iris-versicolor) = 0.0$

$P(Iris-virginica) = 0.0$

## 十七、ROC 与 AUC

代码中计算了三个类别各自的 ROC 和 AUC：

```python
y_predict_proba = algo.predict_proba(x_train)

y1_true = (y_train == 0).astype(np.int)
y1_score = y_predict_proba[:, 0]
fpr1, tpr1, _ = roc_curve(y1_true, y1_score)
auc1 = auc(fpr1, tpr1)

y2_true = (y_train == 1).astype(np.int)
y2_score = y_predict_proba[:, 1]
fpr2, tpr2, _ = roc_curve(y2_true, y2_score)
auc2 = auc(fpr2, tpr2)

y3_true = (y_train == 2).astype(np.int)
y3_score = y_predict_proba[:, 2]
fpr3, tpr3, _ = roc_curve(y3_true, y3_score)
auc3 = auc(fpr3, tpr3)

print((auc1, auc2, auc3))
```

输出：

```
(1.0, 0.9981684981684982, 0.9981684981684982)
```

AUC 可以理解为模型区分正负样本的能力：

- $AUC = 1$ 表示完美区分；
- $AUC = 0.5$ 表示和随机猜差不多；
- $AUC < 0.5$ 通常说明模型判断方向可能反了。

本例中三个类别的 AUC 都非常接近 $1$，说明模型对三类鸢尾花的区分能力很强。

因为这是三分类问题，所以代码采用“一类对其他类”的方式分别计算：

- 类别 0 vs 其他类别；
- 类别 1 vs 其他类别；
- 类别 2 vs 其他类别。

例如：

```python
y1_true = (y_train == 0).astype(np.int)
```

这行代码的意思是：

- 如果样本真实类别是 0，就记为 1；
- 否则记为 0。

这样就把多分类问题临时转成了二分类问题，方便计算 ROC 曲线。

## 十八、ROC 曲线绘制

绘图代码如下：

```python
plt.plot(fpr1, tpr1, 'r-o')
plt.plot(fpr2, tpr2, 'g-o')
plt.plot(fpr3, tpr3, 'b-o')
plt.show()
```

其中：

| 变量     | 含义          |
| -------- | ------------- |
| `fpr`    | 假正例率      |
| `tpr`    | 真正例率      |
| 红色曲线 | 类别 0 的 ROC |
| 绿色曲线 | 类别 1 的 ROC |
| 蓝色曲线 | 类别 2 的 ROC |

真正例率公式：

$TPR = \frac{TP}{TP + FN}$

假正例率公式：

$FPR = \frac{FP}{FP + TN}$

ROC 曲线越靠近左上角，模型越好。
本例 AUC 接近 $1$，因此曲线整体表现非常优秀。

| `node #0`                 | 节点编号                 |
| `x3 <= 2.35`              | 当前节点的判断条件       |
| `gini = 0.665`            | 当前节点的基尼系数       |
| `samples = 120`           | 当前节点包含 120 个样本  |
| `value = [36, 42, 42]`    | 三个类别的样本数量       |
| `class = Iris-versicolor` | 当前节点中数量最多的类别 |

根节点含义是：

- 当前一共有 120 个训练样本；
- 其中类别 0 有 36 个；
- 类别 1 有 42 个；
- 类别 2 有 42 个；
- 节点还比较混杂，所以 $gini = 0.665$。

第一层判断是：

$x_3 \leq 2.35$

如果条件为真，则进入左子节点。左子节点显示：

```
gini = 0.0
samples = 36
value = [36, 0, 0]
class = Iris-setosa
```

这说明，只要满足这个条件的 36 个样本全部都是 `Iris-setosa`，没有混杂样本，所以基尼系数为：

$Gini = 0$

这也是决策树非常直观的地方：
它发现“花瓣长度较短”的样本几乎可以直接判断为山鸢尾。

如果第一层条件不满足，则继续看其他条件，例如：

$x_4 \leq 1.65$

后续节点主要继续用花瓣长度和花瓣宽度区分 `Iris-versicolor` 和 `Iris-virginica`。

这也和特征重要性结果对应：

```
[0.         0.         0.55131392 0.44868608]
```

说明这棵树基本只依赖花瓣相关特征完成分类。

## 二十一、为什么决策树适合入门

决策树的优点非常明显：

1. 可解释性强：可以画出树图，看到每一步判断；
2. 不需要复杂的数据预处理；
3. 可以处理分类问题，也可以处理回归问题；
4. 对特征尺度不敏感；
5. 非线性能力较强，能学习复杂边界。

不过它也有缺点：

1. 容易过拟合；
2. 对数据扰动比较敏感；
3. 单棵树的稳定性不如随机森林、梯度提升树；
4. 如果树太深，规则会变得复杂，不容易泛化。

因此实际项目中，决策树常常作为理解模型原理的第一站。
如果追求更强效果，可以进一步学习：

- 随机森林；
- AdaBoost；
- GBDT；
- XGBoost；
- LightGBM；
- CatBoost。

这些算法本质上都和树模型有密切关系。

## 二十二、完整流程总结

本案例的机器学习流程可以总结为：

```
读取数据
  ↓
拆分特征 X 和标签 Y
  ↓
标签编码
  ↓
划分训练集和测试集
  ↓
创建决策树分类器
  ↓
训练模型
  ↓
输出特征重要性
  ↓
评估训练集和测试集效果
  ↓
计算 ROC 和 AUC
  ↓
导出决策树可视化图
```

对应到代码核心步骤：

```python
df = pd.read_csv(path, sep=",", header=None, names=['A', 'B', 'C', 'D', 'cla'])

X = df.iloc[:, :4]
X = np.asarray(X).astype(np.float64)

Y = df['cla']
label_encoder = LabelEncoder()
Y = label_encoder.fit_transform(Y)

x_train, x_test, y_train, y_test = train_test_split(
    X,
    Y,
    test_size=0.2,
    random_state=2
)

algo = DecisionTreeClassifier(max_depth=3)
algo.fit(x_train, y_train)

print(algo.feature_importances_)
print(classification_report(y_test, algo.predict(x_test)))
print(algo.score(x_test, y_test))
```

## 二十三、应用场景

决策树不仅能用于鸢尾花分类，还可以应用在很多实际业务中。

| 场景     | 示例                           |
| -------- | ------------------------------ |
| 医疗诊断 | 根据检查指标判断疾病风险       |
| 金融风控 | 判断贷款用户是否可能违约       |
| 用户运营 | 判断用户是否可能流失           |
| 电商推荐 | 判断用户是否可能购买某商品     |
| 教育评估 | 根据学习行为预测考试通过率     |
| 工业检测 | 根据传感器数据判断设备是否异常 |

比如在贷款审批中，决策树可能学到类似规则：

```
如果收入较高，并且历史还款记录良好，则通过；
如果收入较低，并且逾期次数较多，则拒绝；
否则继续查看负债率、工作年限等特征。
```

这种规则非常适合给业务人员解释，因此决策树常用于需要模型可解释性的场景。
