# GBDT梯度提升树

GBDT梯度提升树——用"纠错接力"逼近完美预测

## 概念介绍

想象你在射箭比赛中瞄准靶心：

- 第一箭偏了 3 厘米 → 你记住这个偏差
- 第二箭专门瞄准"偏差方向" → 偏差缩小到 1 厘米
- 第三箭继续瞄准"剩余偏差" → 几乎命中靶心

**GBDT（Gradient Boosting Decision Tree，梯度提升决策树）** 就是这样一种"纠错接力"算法：

- 先用一个简单值（如均值）做初始预测
- 计算预测与真实值之间的**残差**（偏差）
- 训练一棵新的决策树去拟合这个残差
- 把新树的预测"加"到原来的预测上
- 重复以上过程，残差越来越小，预测越来越准

和 AdaBoost 的区别：AdaBoost 通过调整**样本权重**来关注难样本，GBDT 通过拟合**残差（负梯度）** 来逐步修正误差。

## 原理讲解

### 核心机制

GBDT 属于**前向加法模型**——每一步在已有模型基础上"加"一棵新树：

$$F_m(x) = F_{m-1}(x) + \nu \cdot f_m(x)$$

其中 $\nu$ 是学习率（步长），$f_m$ 是第 $m$ 棵树。

### 步骤拆解

| 步骤        | 操作                                  | 类比                 |
| ----------- | ------------------------------------- | -------------------- |
| 1. 初始化   | $F_0 = \bar{y}$（用均值做第一次预测） | 先蒙一个平均分       |
| 2. 计算残差 | $r_i = y_i - F_{m-1}(x_i)$            | 看看离靶心还差多远   |
| 3. 拟合残差 | 训练一棵树 $f_m$ 去预测残差           | 让新队友专门补偿偏差 |
| 4. 更新模型 | $F_m = F_{m-1} + \nu \cdot f_m$       | 把修正量叠加上去     |
| 5. 重复     | 回到步骤 2，直到残差足够小            | 不断精修，逼近完美   |

### 为什么叫"梯度"提升？

在平方损失函数 $L = \frac{1}{2}(y - F(x))^2$ 下：

$$-\frac{\partial L}{\partial F(x)} = y - F(x)$$

负梯度恰好等于残差！所以"拟合残差"本质上就是在"沿着梯度方向下降"。对于其他损失函数（如绝对值损失、对数损失），负梯度不再是简单残差，但思想一致。

## 公式推导

### 1. 初始化

$$F_0(x) = \bar{y} = \frac{1}{N}\sum_{i=1}^{N} y_i$$

对于本例：$F_0 = \frac{5.56 + 5.70 + \cdots + 9.05}{10} = 7.307$

### 2. 第 $m$ 轮残差（负梯度）

$$r_{m,i} = y_i - F_{m-1}(x_i)$$

- $y_i$：真实值
- $F_{m-1}(x_i)$：前 $m-1$ 轮的累计预测值

### 3. 拟合残差树

训练决策树 $f_m(x)$，使其尽可能拟合 $\{(x_i, r_{m,i})\}$。

### 4. 模型更新

$$F_m(x) = F_{m-1}(x) + \nu \cdot f_m(x)$$

- $\nu$：学习率，控制每棵树的贡献大小（防止步子太大过拟合）
- 本例中 $\nu = 1$，即完全信任每棵树的修正

### 5. 最终模型

$$F_M(x) = F_0 + \sum_{m=1}^{M} \nu \cdot f_m(x)$$

## 代码逻辑

### 数据准备

```python
import pandas as pd
import numpy as np
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import r2_score

df = pd.DataFrame([[1, 5.56], [2, 5.7], [3, 5.91], [4, 6.4], [5, 6.8],
                   [6, 7.05], [7, 8.9], [8, 8.7], [9, 9], [10, 9.05]],
                  columns=['X', 'Y'])

X = df.iloc[:, :-1]
Y = df.iloc[:, -1]
y = Y  # 保留原始标签用于最终评估
```

10 个样本，特征是 1~10 的数字，标签是对应的连续值——典型的回归问题。

### 训练过程：逐步拟合残差



```python
# 初始化：用均值作为第一次预测
F0 = np.mean(Y)           # F0 = 7.307
M = [F0]                  # 模型列表，第一个元素是初始值

Y = Y - F0               # 第一轮残差 = 真实值 - 均值
n_trees = 10             # 迭代10棵树
learning_rate = 1        # 学习率为1（完全修正）

for i in range(n_trees):
    model = DecisionTreeRegressor(max_depth=1)  # 决策树桩
    model.fit(X, Y)                             # 拟合当前残差
    Y = Y - learning_rate * model.predict(X)    # 更新残差 = 旧残差 - 本轮预测
    M.append(model)                             # 保存本轮模型
```

每轮循环做了三件事：训练树 → 预测残差 → 残差再缩小。10 轮后，残差已经非常接近 0。

### 预测过程：逐层叠加

res = np.zeros(df.shape[0])
for j in range(len(M)):

        res += M[j]                        # 加上初始均值

## 总结

1. **核心思想**：GBDT 通过逐轮拟合残差（负梯度），让每棵新树专门修正上一轮的预测偏差，多棵简单树"接力"逼近真实值
2. **关键参数**：树的数量（迭代轮次）和学习率需要平衡——学习率小则需要更多树，但泛化更好；学习率大则收敛快但容易过拟合
3. **工程价值**：GBDT 是 XGBoost、LightGBM、CatBoost 等工业级框架的理论基础，理解它就掌握了 Boosting 家族的核心脉络
