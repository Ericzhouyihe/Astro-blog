## 信息熵

信息熵用于衡量一个随机变量的不确定性。

在决策树中，如果一个数据集中的类别越混乱、不确定性越高，那么它的信息熵就越大；如果数据集越纯，信息熵就越小。

信息熵公式为：

$$
H(X) = -\sum_{i=1}^{n} p_i \log_2 p_i
$$

其中：

- $p_i$ 表示第 $i$ 类样本出现的概率
- $\log_2$ 表示以 2 为底的对数
- 信息熵的单位通常是 bit

如果某一类的概率为 0，则这一项不参与计算，因为：

$$
\lim_{p \to 0} p \log p = 0
$$

## 代码实现

使用 `numpy` 实现了信息熵计算：

```python
import numpy as np

def entropy(p):
    return np.sum([-t * np.log2(t) for t in p if t != 0])
```

这里的 `p` 是一个概率分布列表，例如 `[0.5, 0.5]`。

代码中的核心逻辑是：

```
-t * np.log2(t)
```

也就是信息熵公式中的：

$ -p_i \log_2 p_i$

## 信息熵示例

当两个类别概率相等时：

```
print(entropy([0.5, 0.5]))
```

输出结果：

```
1.0
```

说明二分类且概率完全相等时，不确定性为 1。

当四个类别概率完全相等时：

```
print(entropy([0.25, 0.25, 0.25, 0.25]))
```

输出结果：

```
2.0
```

四分类且概率均匀分布时，不确定性更高，因此信息熵更大。

## 概率分布对信息熵的影响

比较三组概率分布：

```
print(entropy([0.25, 0.25, 0.25, 0.25]))
print(entropy([0.65, 0.25, 0.05, 0.05]))
print(entropy([0.97, 0.01, 0.01, 0.01]))
```

输出结果：

```
2.0
1.3361602543738118
0.24194073285321088
```

可以看到：

- `[0.25, 0.25, 0.25, 0.25]` 最均匀，信息熵最大
- `[0.65, 0.25, 0.05, 0.05]` 有明显倾向，信息熵降低
- `[0.97, 0.01, 0.01, 0.01]` 几乎确定属于第一类，信息熵很小

因此可以总结：

> 概率分布越均匀，信息熵越大；概率分布越集中，信息熵越小。

## 条件熵

条件熵表示在已知某个条件之后，随机变量仍然存在的不确定性。

在决策树中，条件熵常用来衡量按照某个特征划分数据集之后，类别标签还剩下多少不确定性。

条件熵公式为：

$ H(Y|X) = \sum_{i=1}^{n} P(X=x_i)H(Y|X=x_i)$

其中：

- $X$ 表示某个特征
- $Y$ 表示类别标签
- $H(Y|X)$ 表示在已知特征 $X$ 后，类别 $Y$ 的不确定性
- $P(X=x_i)$ 是特征取值为 $x_i$ 的概率
- $H(Y|X=x_i)$ 是在该特征取值下，类别标签的信息熵

## 条件熵计算示例

条件熵计算例子：

```
# x1=是 p1=0.7   Y：5:2
# x1=否 p2=0.3   Y：1:2
# H(Y|x1) = p1*entropy([2/7,5/7]) + p2*entropy([1/3,2/3])

print(0.7*entropy([2/7,5/7]) + 0.3*entropy([1/3,2/3]))
```

输出结果：

```
0.8796731482129885
```

这个计算过程可以理解为：

当特征 `x1=是` 时，该分支占总体的比例为 `0.7`，类别分布为：[2/7, 5/7]

当特征 `x1=否` 时，该分支占总体的比例为 `0.3`，类别分布为：[1/3, 2/3]

所以条件熵为：

$ H(Y|x_1) = 0.7H([2/7,5/7]) + 0.3H([1/3,2/3])$

最终结果为：

$ H(Y|x_1) = 0.8797$

## 条件熵和特征选择的关系

在决策树中，我们希望选择一个特征，使得划分之后的数据集更加纯。

也就是说：

- 划分前，类别的不确定性较高
- 使用某个特征划分后，类别的不确定性降低
- 降低得越多，说明这个特征越有区分能力

这就引出了**信息增益**：

$ Gain(Y, X) = H(Y) - H(Y|X)$

其中：

- $H(Y)$ 是划分前的整体信息熵
- $H(Y|X)$ 是按照特征 $X$ 划分后的条件熵
- 信息增益越大，说明该特征越适合作为决策树的划分特征

因此，在 ID3 决策树算法中，通常会选择信息增益最大的特征进行划分。

## 总结

信息熵衡量的是数据本身的不确定性。

条件熵衡量的是在已知某个特征之后，类别标签仍然存在的不确定性。

在决策树特征选择中，一个好的特征应该能够显著降低类别的不确定性，也就是让条件熵尽可能小，让信息增益尽可能大。

简单来说：

> 信息熵看整体混乱程度，条件熵看按特征划分后的混乱程度，信息增益看这个特征能让数据变得多清晰。
