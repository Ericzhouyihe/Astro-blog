# 1. PCA 是什么？

PCA（Principal Component Analysis，主成分分析）是经典的**无监督线性降维算法**，核心作用是在尽可能保留原始信息的前提下，将高维数据映射到低维空间。

它并非直接筛选丢弃原始特征，而是对所有原始特征做线性重组，生成一组彼此正交（线性无关）的新特征，这些新特征就叫做主成分。

# 2. 为什么需要降维？

高维数据通常会带来四类问题：
1. **计算成本高**：特征越多，模型训练和推理的时间、内存消耗越大
2. **容易过拟合**：特征数量远大于样本数时，模型易学到噪声而非数据规律
3. **可视化困难**：人类仅能直观理解二维、三维数据
4. **信息冗余**：很多特征高度相关，携带的信息存在重复

PCA 的目标就是用更少的维度保留核心信息，同时消除特征间的冗余。

# 3. PCA 的核心直觉

以二维降一维的场景举例：已知学生的数学、物理两门成绩，想用一个数值代表学生的综合学习能力。

直接丢弃一门成绩会损失大量信息，更合理的方式是构造加权综合分：
```text
综合分 = a × 数学成绩 + b × 物理成绩
```

PCA 的做法就是自动寻找最优权重，让投影后的综合分最大程度区分不同样本，这个综合分就是第一主成分（PC1）。

# 4. 什么是“信息最多”？

PCA 用**方差**来衡量信息量：
- 方差大：数据分布更分散，样本区分度高
- 方差小：数据分布集中，样本区分度低

因此，第一主成分 PC1，就是让数据投影后方差最大的方向。
如果需要第二个主成分 PC2，则会在与 PC1 正交的所有方向里，选择方差最大的那一个。

PC3则需要同时满足与前两个都保持正交的基础上，选择方差最大的一个。后续依次需要和前面的全部正交。

# 5. PCA 的数学步骤

设原始数据矩阵 `X` 形状为 `m × n`（`m` 为样本数，`n` 为原始特征数），PCA 的目标是得到降维后矩阵 `Y`（形状 `m × k`，`k << n`）。

## 5.1 数据标准化
PCA 对特征尺度非常敏感，通常先做 Z-Score 标准化：
$x' = \frac{x - \mu}{\sigma}$
其中 $\mu$ 是特征均值，$\sigma$ 是特征标准差。标准化后各特征均值为 0、标准差为 1，避免数值范围大的特征主导主成分方向。

## 5.2 计算协方差矩阵
协方差衡量两个变量的线性相关程度，标准化后协方差矩阵为：
$C = \frac{1}{m-1}X^TX$

## 5.3 特征值与特征向量分解
对协方差矩阵做特征分解：
$Cv = \lambda v$
- 特征值 $\lambda$：对应主成分的方差，值越大说明该主成分保留的信息越多
- 特征向量 $v$：对应主成分的方向

## 5.4 选择前 k 个主成分
将特征值从大到小排序：$\lambda_1 \geq \lambda_2 \geq ... \geq \lambda_n$，选取前 `k` 个最大特征值对应的特征向量，组成 `n × k` 的投影矩阵 `W`。

## 5.5 投影得到降维数据
将标准化后的原始数据投影到新空间：
$Y = XW$

# 6. 如何选择 k 值？

`k` 是最终保留的主成分数量，常用两种选择方法：

## 6.1 累计方差贡献率

单个主成分的方差贡献率：$\frac{\lambda_i}{\sum_{j=1}^{n}\lambda_j}$
前 `k` 个主成分的累计方差贡献率：$\frac{\sum_{j=1}^{k}\lambda_j}{\sum_{j=1}^{n}\lambda_j}$

实际使用中，通常选择最小的 `k`，使累计方差贡献率达到 **85% ~ 95%**。

## 6.2 肘部法则
绘制“主成分序号 - 特征值”折线图，曲线由陡变缓的“肘部”就是分界点：肘部之前的主成分信息贡献大，肘部之后贡献微弱，可以丢弃。

# 7. PCA 的优点

1. **无监督**：不需要标签，可直接处理无标注数据
2. **降本提效**：减少特征数量，加快模型训练与推理速度
3. **缓解过拟合**：丢弃低方差信息，降低模型学习噪声的风险
4. **消除共线性**：主成分彼此正交，可缓解特征多重共线性问题
5. **辅助去噪**：噪声多存在于低方差方向，丢弃后可实现一定去噪效果

# 8. PCA 的缺点

1. **仅支持线性降维**：对存在非线性结构的数据效果有限
2. **对异常值敏感**：异常值会干扰协方差矩阵，改变主成分方向
3. **可解释性弱**：主成分是原始特征的线性组合，物理含义不直观
4. **必然损失信息**：只要降维，就不可避免会丢失部分细节

# 9. PCA 的常见变体

| 变体            | 作用                         |
| --------------- | ---------------------------- |
| Kernel PCA      | 借助核函数实现非线性降维     |
| Incremental PCA | 支持分批训练，适配大规模数据 |
| Sparse PCA      | 生成稀疏主成分，提升可解释性 |
| Robust PCA      | 增强对异常值、噪声的鲁棒性   |

# 10. PCA 的典型应用

1. **图像压缩**：用少量主成分表示图像，降低存储开销
2. **人脸识别**：经典 Eigenfaces 方法基于 PCA 思想实现
3. **数据可视化**：高维数据降至二维/三维，直观观察数据分布
4. **机器学习预处理**：分类/回归前降维，提升训练效率
5. **异常检测**：低方差主成分中可挖掘异常特征
6. **推荐系统**：对用户-物品矩阵降维，提取潜在特征

# 11. 常见误区和总结

很多人误以为 PCA 是“从原始特征里挑几个重要的，丢掉其他的”，这是错误的。

PCA 不是挑选原始特征，而是**重组原始特征生成全新的主成分**。比如 100 维特征降到 20 维，是生成 20 个复合特征，每个主成分都可能包含全部 100 个原始特征的信息

PCA 的本质是：找到数据中方差最大的若干正交方向，将原始数据投影到这些方向上，以更少的维度保留尽可能多的信息。

通俗来说，PCA 不是扔掉特征，而是把原始特征重新组合，提炼出更少、更核心的“超级特征”。

# 12. Python 实现 PCA

使用 `scikit-learn` 可快速实现 PCA：
```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# X 为原始数据，形状 (m, n)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 保留 95% 方差，自动选择主成分数量
pca = PCA(n_components=0.95)
Y = pca.fit_transform(X_scaled)

print(f"降维后维度：{Y.shape[1]}")
print(f"累计方差贡献率：{sum(pca.explained_variance_ratio_):.2f}")
```

# 13. 举例总结

## 13.1 假设有 3 个特征

- \(x_1\)：身高
- \(x_2\)：体重
- \(x_3\)：肺活量

这 3 个特征一起，构成一个**三维特征空间**。

## 13.2 PC1 在干什么？

它不是只看身高、或只看体重，
而是在**这 3 个特征组合出来的整个空间里**，
找一条**直线方向**，使得：

> 所有数据点投影到这条直线上后，**方差最大、分布最开**。

## 13.3 数学上怎么表达这个方向？

PC1 本质是一个**系数向量**：
$PC_1 = a_1 x_1 + a_2 x_2 + a_3 x_3$

- $a_1$, $a_2$, $a_3$ 是权重（载荷 loading）
- 它是**所有特征的线性组合**
- 目标：让组合后的新变量方差最大

## 13.4 找PC2，PC3，...，PCn

找PC2：

- 仍然在全部的3个特征中找
- 但必须在和PC1正交（垂直）的方向里，然后选出方差最大的一条直线

找PC3：

- 仍然在全部的3个特征中找
- 必须和PC1，PC2都正交，然后选出方差最大的一条直线

## 13.5选择前k个主成分进行降维

信息多少 = 方差大小

PCA 里一句话：

**方差越大 = 信息越多**

为什么？

- 数据差别越大，说明**区分度强、有规律、有信息量**
- 数据挤在一起、抖动很小，就是**噪声、没用信息**

所以 PCA 直接用**方差占比（方差贡献率）**衡量信息量：

- PC1 方差占比：比如 85%
- PC2 方差占比：比如 12%
- PC3 方差占比：比如 3%

加起来 = 100%（全部信息）



但是特征过多之后，就会导致模型变得很复杂，就会导致出现背答案的过拟合问题

当有100个特征的时候，能算出100个PC，但前 10 个 PC 就占了 95% 信息

这时可以**直接丢掉后面 90 个 PC**，结果：**100 维 → 10 维**，模型训练更快、更稳、不卡。
