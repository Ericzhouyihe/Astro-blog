# 特征选择

特征选择的目标很简单：从原始特征里，挑出对模型最有用的那一部分。

它的好处主要有三个：

1. 减少维度，训练更快
2. 去掉噪声特征，提升模型效果
3. 降低过拟合风险，让模型更稳

常见的特征选择方法可以分成三类：

- 过滤式（Filter）
- 包装式（Wrapper）
- 嵌入式（Embedded）

下面用 sklearn 里的几个常用方法来看看。

## 1. 示例数据

```python
import numpy as np
from sklearn.feature_selection import VarianceThreshold, SelectKBest
from sklearn.feature_selection import f_regression, chi2
from sklearn.feature_selection import RFE
from sklearn.feature_selection import SelectFromModel
from sklearn.linear_model import LogisticRegression

X = np.array([
    [0, 2, 0, 3],
    [0, 1, 4, 3],
    [0.1, 1, 1, 3],
    [1, 2, 3, 1],
    [2, 3, 4, 3]
], dtype=np.float32)

Y = np.array([1, 2, 1, 2, 1])
```

这里的 `X` 有 4 个特征，`Y` 是标签。
我们的目标就是判断这 4 个特征里，哪些更值得保留。

------

## 2. 方差选择法

### 思路

一个特征如果变化特别小，说明它几乎没什么区分能力。
比如某一列几乎所有样本都一样，这种特征通常可以直接删掉。

### 代码

```
variance = VarianceThreshold(threshold=0.7)
variance.fit(X)

print(variance.variances_)
print(variance.transform(X))
```

### 结果理解

输出的方差是：

```
[0.6176 0.56   2.64   0.64]
```

阈值设为 `0.7`，所以只有第 3 个特征的方差大于阈值，会被保留下来。

### 特点

- 优点：简单、快
- 缺点：只看“波动大小”，不看和标签的关系

------

## 3. 相关系数法

### 思路

如果一个特征和目标值关系更强，那它通常更有用。
`SelectKBest` 就是“只保留得分最高的 K 个特征”。

这里用的是 `f_regression`，它会给每个特征打一个相关性分数。

### 代码

```
sk1 = SelectKBest(f_regression, k=2)
sk1.fit(X, Y)

print(sk1.scores_)
print(sk1.transform(X))
```

### 结果理解

分数大致是：

```
[0.04736842 0.36       1.32       1.8       ]
```

分数最高的两个特征是第 4 列和第 3 列，所以最后保留这两列。

### 特点

- 优点：简单直观
- 缺点：每个特征是单独打分的，没考虑特征之间的组合关系

------

## 4. 卡方检验

### 思路

卡方检验常用于分类任务，判断某个特征和类别之间是否有关系。
它更适合非负特征，比如词频、计数这类数据。

### 注意

`chi2` 要求特征值必须是非负数。
如果特征里有负数，先做平移或换别的方法。

### 代码

```
sk2 = SelectKBest(chi2, k=2)
sk2.fit(X, Y)

print(sk2.scores_)
print(sk2.transform(X))
```

### 结果理解

分数最高的两个特征同样会被保留下来。
这个例子里保留的是第 3 列和第 4 列。

### 特点

- 优点：适合分类问题
- 缺点：要求输入非负

------

## 5. Wrapper - 递归特征消除法

### 思路

包装式方法会“训练模型，再删特征，再训练模型”，反复迭代。
RFE 就是其中最经典的一种。

它的逻辑是：

1. 先训练一个模型
2. 根据模型系数或重要性，删掉一部分最差的特征
3. 继续训练
4. 直到剩下指定数量的特征

### 代码

```
estimator = LogisticRegression()
selector = RFE(estimator, step=2, n_features_to_select=3)
selector.fit(X, Y)

print(selector.support_)
print(selector.ranking_)
print(selector.transform(X))
```

### 结果理解

输出：

```
[False  True  True  True]
[2 1 1 1]
```

说明第 1 个特征被删掉了，剩下第 2、3、4 个特征。

### 特点

- 优点：考虑了模型效果，通常比过滤法更强
- 缺点：计算量大，速度慢

------

## 6. Embedded - 基于模型的特征选择

### 思路

嵌入式方法是在训练模型的过程中，顺便完成特征筛选。
最常见的就是带正则化的线性模型。

这里用的是：

```
SelectFromModel(LogisticRegression(penalty='l2'))
```

它会根据模型系数大小来判断哪些特征更重要。

### 代码

```
X2 = np.array([
    [ 5.1,  3.5,  1.4,  0.2],
    [ 4.9,  3. ,  1.4,  0.2],
    [ -6.2,  0.4,  5.4,  2.3],
    [ -5.9,  0. ,  5.1,  1.8]
], dtype=np.float64)

Y2 = np.array([0, 0, 2, 2])

estimator = LogisticRegression(penalty='l2', C=0.1)
sfm = SelectFromModel(estimator, threshold=0.09)
sfm.fit(X2, Y2)

print(sfm.transform(X2))
print(sfm.estimator_.coef_)
```

### 结果理解

模型系数大致是：

```
[[-0.28319012 -0.07821437  0.09858841  0.04716025]]
```

按阈值 `0.09` 来看，保留下来的特征是第 1 列和第 3 列。

### 特点

- 优点：效率高，和模型结合紧密
- 缺点：结果依赖具体模型

------

## 7. 四种方法怎么选

可以简单记成这样：

- 方差选择法：先删掉“几乎没变化”的特征
- 相关系数法：选和标签关系强的特征
- RFE：一边训练一边删，效果通常更强
- SelectFromModel：直接利用模型本身做筛选

------

## 8. 小结

特征选择不是越多越好，而是越合适越好。

一句话总结这节内容：

- 过滤式：快，适合先筛一遍
- 包装式：准，但慢
- 嵌入式：实用，常和模型一起用

实际项目里，常见做法是先用简单方法粗筛，再用模型方法精筛。
