## 一、什么是优先队列

### 1.1 通俗理解
普通队列遵循 **先进先出（FIFO）** 原则，就像日常排队，先来的人先办事。
优先队列不遵循入队顺序，而是遵循 **优先级高者先出**，类似医院急诊：重症患者无论来的早晚，永远优先接诊。

### 1.2 正式定义
优先队列是一种**抽象数据类型（ADT）**，核心支持两个操作：
1. **插入元素**：向集合中加入一个带优先级的元素
2. **取出极值**：快速获取并删除当前优先级最高（最大/最小）的元素

在工程与算法面试中，**优先队列几乎都用「二叉堆」实现**，因此入门阶段可以近似认为：
> 优先队列 ≈ 堆

## 二、核心特性与时间复杂度

优先队列的核心价值：**在动态变化的数据中，高效维护最大值或最小值**。

| 操作         | 普通数组遍历 |  全程排序  | 优先队列（二叉堆） |
| :----------- | :----------: | :--------: | :----------------: |
| 查看当前极值 |     O(n)     |    O(1)    |      **O(1)**      |
| 插入新元素   |     O(1)     | O(n log n) |    **O(log n)**    |
| 删除极值     |     O(n)     | O(n log n) |    **O(log n)**    |
| 批量建堆     |      -       | O(n log n) |      **O(n)**      |

**适用场景判断**：当你需要**反复多次「取极值 + 插入新元素」**时，优先队列的效率会远高于每次重新排序。如果只是静态数据取一次极值，直接遍历或排序即可。

## 三、底层原理：二叉堆详解

### 3.1 二叉堆的本质
二叉堆是一颗**完全二叉树**，且满足堆性质：
- **大顶堆**：每个父节点的值 ≥ 左右子节点的值，堆顶是全局最大值
- **小顶堆**：每个父节点的值 ≤ 左右子节点的值，堆顶是全局最小值

由于是完全二叉树，二叉堆可以**用数组直接存储**，无需链表。假设数组下标从 0 开始：
- 下标为 `i` 的节点，**左孩子**下标为 `2*i + 1`
- 下标为 `i` 的节点，**右孩子**下标为 `2*i + 2`
- 下标为 `i` 的节点，**父节点**下标为 `(i - 1) // 2`

### 3.2 两个核心操作
#### ① 上浮（sift up）——插入元素时使用
1. 把新元素放到数组末尾（堆的最底层）
2. 不断和父节点比较：如果比父节点更“优”（大顶堆就更大，小顶堆就更小），就和父节点交换位置
3. 直到到达堆顶，或不再满足交换条件

时间复杂度：O(log n)，最多从树底走到树顶。

#### ② 下沉（sift down）——删除堆顶时使用
1. 把数组最后一个元素移到堆顶（覆盖原堆顶）
2. 不断和左右孩子比较：和两个孩子中更“优”的那个交换位置
3. 直到到达叶子节点，或比两个孩子都更“优”

时间复杂度：O(log n)，最多从树顶走到树底。

### 3.3 堆化（Heapify）
将一个无序数组直接调整成堆，时间复杂度 **O(n)**，比一个个插入元素（O(n log n)）效率更高。
做法：从最后一个非叶子节点开始，从后往前依次执行下沉操作。

## 四、主流语言内置 API 用法

面试和工程中**不需要手写堆**，直接调用语言内置 API 即可。

### 4.1 Python（heapq 模块）
Python 内置的 `heapq` 只实现了**小顶堆**，要实现大顶堆，存入元素的相反数即可。

```python
import heapq

# 1. 创建空堆
heap = []

# 2. 插入元素 O(log n)
heapq.heappush(heap, 3)
heapq.heappush(heap, 1)
heapq.heappush(heap, 2)

# 3. 弹出最小值 O(log n)
min_val = heapq.heappop(heap)  # 返回 1

# 4. 查看堆顶（最小值） O(1)
print(heap[0])  # 输出 2

# 5. 列表原地堆化 O(n)
nums = [5, 2, 7, 1, 9]
heapq.heapify(nums)

# 6. 大顶堆技巧：存负数
max_heap = []
heapq.heappush(max_heap, -3)
heapq.heappush(max_heap, -1)
max_val = -heapq.heappop(max_heap)  # 返回 3
```

### 4.2 Java（PriorityQueue）
默认是小顶堆，大顶堆需要传入比较器。

```java
import java.util.PriorityQueue;

// 小顶堆（默认）
PriorityQueue<Integer> minHeap = new PriorityQueue<>();

// 大顶堆
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);

// 常用方法
minHeap.offer(3);    // 插入
minHeap.poll();      // 弹出最小值
minHeap.peek();      // 查看最小值
```

### 4.3 C++（priority_queue）
默认是大顶堆，小顶堆需要传入 `greater`。

```cpp
#include <queue>
using namespace std;

// 大顶堆（默认）
priority_queue<int> maxHeap;

// 小顶堆
priority_queue<int, vector<int>, greater<int>> minHeap;

// 常用方法
maxHeap.push(3);    // 插入
maxHeap.pop();      // 弹出最大值
maxHeap.top();      // 查看最大值
```

## 五、经典应用场景与解题模板

### 场景 1：Top K 问题
**题目特征**：找第 K 大元素、前 K 个高频元素、前 K 小数字
**标准思路**：求前 K 大用**小顶堆**，求前 K 小用**大顶堆**，堆的大小固定为 K。

**例题：LeetCode 215. 数组中的第 K 个最大元素**
```python
import heapq

def findKthLargest(nums, k):
    heap = []
    for num in nums:
        heapq.heappush(heap, num)
        # 堆大小超过k，弹出最小的，保证堆里始终是当前前k大的数
        if len(heap) > k:
            heapq.heappop(heap)
    return heap[0]  # 堆顶就是第k大的元素
```

### 场景 2：动态取极值（模拟类）
**题目特征**：每一轮取出当前最大/最小值处理，处理完可能产生新元素再放回队列。
**典型题**：最后一块石头的重量、任务调度器

**例题：LeetCode 1046. 最后一块石头的重量**
```python
import heapq

def lastStoneWeight(stones):
    # Python无原生大顶堆，存负数
    heap = [-s for s in stones]
    heapq.heapify(heap)
    
    while len(heap) > 1:
        a = -heapq.heappop(heap)  # 第一重的石头
        b = -heapq.heappop(heap)  # 第二重的石头
        if a != b:
            heapq.heappush(heap, -(a - b))
    return -heap[0] if heap else 0
```

### 场景 3：多路归并
**题目特征**：合并 N 个有序数组/链表，每次从所有队首中选最小的。
**典型题**：LeetCode 23. 合并 K 个升序链表、丑数系列

### 场景 4：贪心算法配套
很多贪心策略的核心是「每次选当前最优选项」，优先队列是最佳载体：
- 哈夫曼编码：每次取两个权重最小的节点合并
- Dijkstra 最短路径：每次取出距离起点最近的节点扩展
- 活动安排、会议室分配问题

## 七、常见误区与避坑

1. **名字带队列但不是队列**：不遵守先进先出原则，只按优先级出队，不要被名称误导。
2. **优先队列不是全有序**：它只保证堆顶是极值，内部其他元素的顺序是不确定的。
3. **Python 没有原生大顶堆**：一律用「存负数、取出来再取反」的技巧，不要手写大顶堆。
4. **不要过度使用堆**：静态数据只取一次极值时，直接 `sort()` 更简单，代码更少。
5. **自定义元素优先级**：元素是对象时，传入比较规则，或把「优先级值 + 元素」组成元组入堆。

---

## 八、一句话总结
优先队列（堆）是**动态维护极值**的最优数据结构。看到 Top K、多路归并、贪心取最优、反复取最大/最小的场景，优先想到它，时间复杂度直接从 O(n²) 或 O(n log n) 优化到 O(n log k)。
